# Report Writing Skill

When asked to produce a written report or document:

## Structure
1. Create an outline first — save it to `/workspace/output/outline.md`.
2. Write the full report section by section.
3. Save the final report to `/workspace/output/report.md`.

## Formatting
- Use Markdown with clear headings, bullet points, and tables where appropriate.
- Include a title, date, and summary at the top.
- Cite sources inline when referencing research results.

## File Management
- Keep intermediate drafts in `/workspace/drafts/`.
- Keep final outputs in `/workspace/output/`.
- Use `git add`, `git commit`, and `git push` to persist work.

## Quality
- Re-read the report after writing to check for consistency.
- Ensure all claims are supported by the research data gathered.
- Keep the tone professional and concise unless the user specifies otherwise.
